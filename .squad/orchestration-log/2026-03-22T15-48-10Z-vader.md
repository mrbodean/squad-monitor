### Design Rationale

**Problem:** Inline `<details>` expansion in Agents tab grid created layout churn and poor UX for agents with long charters.

**Solution:** Dedicated detail pages (hidden at load, revealed via hash) keep the Agents grid clean and responsive. Users can drill into any agent without affecting the grid layout.

**UX Flow:**
1. Click agent card → browser navigates to `#agent/{name}`
2. Script shows agent-detail section, hides Agents tab
3. User reads full charter
4. Click "Back to Agents" → hash clears, grid returns
5. URL bookmarkable for sharing: `...#agent/vader`

## Level-Up 🆙

**Navigation without page reload.** Single-page app pattern (SPA) applied within a static HTML file. Hash routing + CSS `display` toggle = zero latency, zero bundle size added, zero dependency. This is how a minimal SPA works.
