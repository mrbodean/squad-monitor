## Quality Spot-Checks

### Coder (Core Dev)

- [ ] Code uses ESM imports where applicable
- [ ] Error handling present
- [ ] Uses gpt-5.4-mini minimum for code generation / IaC
- [ ] Build/test commands work

### Tester (Tester / QA)

- [ ] Tests cover happy path, edge cases, and error paths
- [ ] No regressions introduced

### DevRel (Docs / DevRel)

- [ ] Output uses direct imperative tone (not "you might want to consider...")
- [ ] No marketing fluff or buzzwords
- [ ] Every command in docs was tested first

### Evaluator (Evals / Quality Baseline)

- [ ] Eval baselines reviewed after prompt changes
- [ ] Quality metrics reported

### Scribe (Historian / Build Scribe)

- [ ] Journal entry uses timestamped table format
- [ ] Level-Up moments captured
- [ ] Honest about failures/pivots
- [ ] Quotes human steering commands

## Completion Gate

After any substantial work session, verify:

- [ ] `> **[AgentName]**` role tag was used
- [ ] `.squad/decisions.md` updated if decisions were made
- [ ] `JOURNAL.md` updated if milestone reached
- [ ] Docs updated if behavior changed
- [ ] Tests run if code changed
- [ ] Response ended with "which roles should have touched this?"
